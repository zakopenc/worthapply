# CodySocial: Integrated Asset Protocol (v3.0)

## 1. Image Generation (Banany/Nano-Banana Integration)
CodySocial is strictly prohibited from using default/placeholder images. All visual assets must be generated by the Gemini Image Generation model to match the specific artistic direction of the scene.

**The Workflow:**
1.  **Prompt Engineering:** Before generation, CodySocial must define the visual style using the following template: 
    *   *Prompt:* "[Visual description of scene] | Style: [Professional/Technical/Blackboard] | Theme: [Brand Theme]"
2.  **API Integration:** Use the `gemini-3.1-flash-image-preview` endpoint to generate high-fidelity assets.
3.  **Post-Processing:** All generated images must be processed by **CodyUI** to ensure color palette cohesion and correct transparency handling (e.g., removal of backgrounds for compositing).

**Example generation command:**
```bash
curl -s -X POST \
  "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash-image-preview:generateContent" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [{
      "parts": [
        {"text": "Professional 3D illustration of a SaaS dashboard interface, dark mode, vibrant green data signals, clean UI, high fidelity, 8k resolution"}
      ]
    }]
  }'
```

## 2. Asset Integration (CodySocial + CodyUI + Banany)
- **CodyUI:** Defines the visual specs (dimensions, color profiles, composition).
- **Banany/Gemini:** Generates the raw assets.
- **CodySocial:** Composites the assets into the Remotion scene, applying necessary filters and motion transitions.

## 3. Mandatory Check
CodySocial must verify the generated image meets the brand standards before embedding it into the video. If the image quality is low, he must regenerate with a more specific prompt. No generic assets allowed.
